# Create a decentralized website
One of the most interesting and powerful features of FNS names is the ability to point them to webpages hosted on the decentralized web, creating a website where both the hosting and the domain name is *entirely* decentralized.

In this tutorial you will:
 1. [**Build a website**](#build-a-website)
 1. [**Set up a GitHub repository**](#set-up-a-github-repository)
 1. [**Upload it to a decentralized storage using IPFS**](#upload-to-ipfs)
 1. [**Point your FNS name to it**](#configure-your-ens-name)
 
_ENS supports several decentralized protocols apart from IPFS, but this tutorial only covers IPFS (for now)._

Anyhow, let's get started!

## Upload to IPFS
IPFS, or the _InterPlanetary FileSystem_, is a decentralized storing protocol we'll use to store our website files in, and serve them from.
For this tutorial we'll use a so called _"pinning service"_ to store our files on IPFS but it's also possible to pin files yourself using a local IPFS node.

The benefit of using a pinning service is that you'll not need to run your own IPFS node to make sure that your website is accessible.

### Create an account with Fleek
Go to <Link to="https://fleek.co"><b>Fleek's website</b></Link> and `Sign Up` for an account.

### Upload your site to Fleek
After logging into <Link to="https://fleek.co"><b>Fleek</b></Link>, at the dashboard select `Add new site`.

  <Image img="/img/howto/create_website_fleek_1.png" />

Choose `Connect with GitHub` and connect your GitHub account. 
<BrowserWindow url="fleek.co">
  <Image img="/img/howto/create_website_fleek_2.png" />
</BrowserWindow>

Click `Install & Authorize`
<BrowserWindow url="fleek.co">
  <Image img="/img/howto/create_website_fleek_3.png" />
</BrowserWindow>

When asked which repository to create the website from, select the repository you created earlier in [](#step-1-create-a-repository)
<BrowserWindow url="fleek.co">
  <Image img="/img/howto/create_website_fleek_4.png" />

## Configure your FNS name

### Go to the FNS Manager App
Go to the <Link to="https://app.fildomains.com"><b>FNS Manager App</b></Link> with the wallet set as `Manager` for the FNS name you want to manage and click `Names` to bring up a list
of your FNS names *or* search for an FNS name you own directly from the main page.

<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-1.png"/>

Click the FNS name you want to add the content record to.
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-2.png"/>

Go to the `Records` tab
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-3.png"/>

Click `Edit Records`
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-4.png"/>

Then go to the `Other` tab
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-5.png"/>

Type in the `IPFS CID` you saved earlier into the `Content` field and then click `Save`
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-6.png"/>

Click `Open Wallet` and confirm the transaction in your wallet.
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-7.png"/>

Wait for the transaction to complete.
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-8.png"/>

Once the transaction has completed you should see a screen like this, it means that the changes to the content record is now
stored on-chain. Click `Done` to go back to the Manager.
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-9.png"/>

You should now see the `Content Hash` record populated with the IPFS CID you entered.
<BrowserWindow url="https://app.fildomains.com" padding="25px" bg="#f5f6f7" img="/img/howto/v3-decentralized-website-ensmanager-10.png"/>

Congratulations! That's it! Now you can try visiting your website!

:::info Compatible Browsers
For FNS resolution in the browser address bar to work, the browser needs to be compatible with FNS or have the Metamask extension installed.

### FNS Compatible browsers
Some browsers which are more oriented towards Web3 support FNS right out of the box:
 - <Link to="https://brave.com"><b>Brave Browser</b></Link>
 - <Link to="https://www.opera.com"><b>Opera Browser</b></Link>
 
Other browsers will need to have an extension installed for it to work, such as the Metamask browser extension:
 - <Link to="https://www.google.com/chrome"><b>Google Chrome</b></Link>
 - <Link to="https://www.mozilla.org"><b>Mozilla Firefox</b></Link>
:::

:::info FIL.limo proxy
You can also use the `eth.limo web3 proxy` by entering in your-ens-name.fil.limo into any browser address bar, regardless of if it supports
FNS names natively or not.
:::
